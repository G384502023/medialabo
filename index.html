「Webコンテンツ作成」の成果

本テーマについて

このテーマの学習目的は、Webコンテンツ作成に必要な技術を体系的に習得することである。HTMLによるWebページの基本的な記述方法を学び、CSSを使ってページのレイアウトを指定する方法を理解する。JavaScriptを用いてインタラクティブで動的なWebページを作成する技術を習得し、プロジェクト演習で外部サーバとの通信処理を実践する。最後に、レポート執筆とフィードバックを通じて学習成果を評価する。

プロジェクト課題について

ユーザーが選んだ都市の天気情報を取得し、表示する。HTMLでは都市選択の「表示」ボタン、結果を表示するテーブルを用意する。JavaScriptでは、ボタンがクリックされると選択された都市のIDを取得し、Axiosを使って指定されたAPIから天気データを取得する仕組みを実装した。APIからのレスポンスが成功すると、天気情報（天気、最高気温、最低気温、湿度、風速、風向）を表示する。これにより、ユーザーが選んだ都市の最新の天気情報を簡単に確認できるようになる。

    • 数当てゲーム

    数当てゲームの主な概要は乱数で1〜10の値を決めて、ユーザーは答えを予想し、数を最大3回入力できる。外れた場合、入力よりも答えが大きいか小さいかを表示する。3回以内に答えの数を当てれば勝ち、そうでなければ負けというゲーム。テキストボックスとボタンが設置され、ユーザはテキストボックスに予想した数を入力し、ボタンを押すと判定される。さらに、正解が出ると祝福メッセージが表示され、間違えるとヒントが与えられる。

    課題3-1では、数当てゲームの正解判定を行う。ユーザーの予想（現在は固定値4）と答えを比較し、コンソールに結果を出力する。予想回数が4回以上ならゲーム終了メッセージ、正解ならお祝いメッセージ、不正解なら残り回数に応じたヒントメッセージを表示する。

    課題4-1では、hantei()メソッドを修正して、コンソール出力からHTMLページに出力するように変更した。予想回数と結果は、それぞれ span#kaisu と span#answer 要素に表示され、p#result 要素にまとめられる。これにより、ユーザーはすべての予想の詳細を直接ページで確認できるようになる。

    課題5-1では、最初のp要素のテキストを削除し、ラベル、テキストボックス、回答ボタンを追加した。次に、すべてのhantei()呼び出しを削除し、回答ボタンのクリックイベントハンドラとしてhanteiを登録した。hantei() 関数内では、yoso = 4 の代わりにテキストボックスの値を使用し、文字列型を整数に変換する処理を追加した。

    • 各授業会の学習事項

    第1回WWW と HTML

    今回学習した内容は、HTMLの基本的な要素の構造とその使用方法についてだ。HTML要素は開始タグ、内容、終了タグで構成され、具体例として段落を示すpタグと強調を示すstrongタグなどがあった。また、改行を示すbrなどの空要素や、属性を使った要素の詳細な設定方法も学んだ。

  • 演習の成果:演習1-4:HTMLのさまざまな要素

    • 第2回 CSS による Web ページの装飾

    CSSはHTML要素を装飾する言語で、HTMLが先に記述され、その後にCSSで装飾を行う。例として、段落の文字色を緑にするには、HTMLのheadにリンク要素を追加し、style.cssファイルにp { color: green; }と記述する。複数要素の装飾はカンマで区切りる。箇条書きの点をなくすにはli { list-style-type: none; }を使用する。特定の要素を装飾するにはclass属性を用いるなどといった見た目を変えることを学んだ。

  • 演習の成果:演習2-3: 演習課題

    • 第3回 JavaScript の基本

    JavaScriptはWebページ上で複雑な機能を実現するプログラミング言語で、ページを開いた直後やボタンを押したときにプログラムが実行され、自動更新やアニメーションなどを可能にする。ブラウザAPIを利用することで、DOMの操作や現在位置の取得、2D・3Dグラフィックスの描画、リアルタイム通信が行うことができる。また、サードパーティAPIを使うことで、Twitterのツイート検索やGoogleマップの埋め込みなど、外部サービスとの連携も可能である。

  • 演習の成果:演習3-6: 演習課題

    • 第4回 DOM と JavaScript

    DOMは、Webページのコンテンツを階層化されたノードとして表現するデータ構造であり、JavaScriptを使って、このDOMを操作することで、ページの要素、属性、テキストにアクセスし、変更を加えたり新しい要素を追加したりすることが可能になる。例えば、documentノードを起点にしてHTML全体のノードを操作でき、これにより動的なWebページを実現することができる。

  • 演習の成果:演習4-2〜4-5: 演習課題

    第5回 イベント処理

    イベントとは、Webページ上で発生するユーザーの操作やシステムの動作のことであり、クリックやキー押下、ブラウザのリサイズ、ページのロード完了、フォームの送信などが例である。これらのイベントに対してJavaScriptで処理を実行することを「イベント処理」と呼ぶ。イベントリスナーを使用して特定のイベントを監視し、イベントが発生したときに指定した関数を呼び出すことで実現する。これにより、インタラクティブで動的なWebページの構築が可能になる。

    工夫点，苦心点，強調点，感想

    本プロジェクトは、Webコンテンツ作成の技術を体系的に習得し、実践することを目的として進められた。主な成果としては、ユーザーが選んだ都市の天気情報をリアルタイムで取得し表示する機能を持つアプリケーションと、数当てゲームを作成した。これらのプロジェクトを通じて、数々の工夫点や苦心点を経験し、多くの学びを得た。まずユーザー体験の向上が挙げられ、数当てゲームでは、ユーザーが予想する数を入力するためのテキストボックスと回答ボタンを用意し、正解が出た場合には祝福メッセージ、外れた場合にはヒントを表示する仕組みを実装した。これにより、ゲームがより楽しいものとなった。また、天気情報表示機能では、Axiosライブラリを用いてAPIからデータをリアルタイムで取得し、取得したデータを動的にHTMLテーブルに表示することで、ユーザーが選択した都市の最新の天気情報を瞬時に確認できるようにした。次に、苦心点としては、APIとの通信エラーが発生することがあった。特に、APIのURLが正しくない場合やサーバー側の問題でデータが取得できないことがあり、これを適切に処理するためにエラーハンドリングのコードを追加した。また、数当てゲームにおいては、ユーザーの入力が無効な場合があり、適切なエラーメッセージを表示することが必要であった。さらに、ブラウザの互換性の問題にも直面した。特に、背景画像やテキストのスタイルが異なるブラウザで異なる表示になり、これを調整するために複数のブラウザでのテストを行った。 強調点としては、Web体験が挙げられる。天気情報の表示や数当てゲームを通じて、単なる静的なページではなく、動的で魅力的な体験を行うことができた。デザインにおいては、CSSを使って視覚的に魅力的なページを作成し、背景画像やテキストのスタイルを工夫することで、ユーザーの視覚的な興味を持たせる工夫を行なった。さらに、数当てゲームでは正解やヒントを提供することで、ユーザーがゲームに対してポジティブなフィードバックを得られるようになった。 感想としては、Webコンテンツ作成における技術の統合と実践の楽しさを実感した。HTML、CSS、JavaScriptを組み合わせて、実際に動作するアプリケーションを作成する過程で、それぞれの技術がどのように連携し、どのような役割があることを学んだ。特に、リアルタイムでのデータ取得や動的なUI更新を実装することで、Webページがユーザーにとって魅力的になるかを理解することができた。また、デバッグやブラウザ互換性の問題に対処することで、問題解決能力が向上し、より効果的なWebコンテンツ作成が可能になったのでとても嬉しく感じた。 このプロジェクトを通じて得た知識と経験は、今後のWeb開発に大いに役立つものであり、さらなる技術の向上の改善に挑戦するための貴重な経験となった。
